## 1. 核心服务层 - RefundService

**文件路径**: `src/main/java/com/example/functioncall/service/RefundService.java`

### 1.1 类定义和依赖注入

```java
@Service
public class RefundService {
    // 使用 SLF4J 记录日志，便于追踪 AI 行为
    private static final Logger log = LoggerFactory.getLogger(RefundService.class);
    
    // Spring AI 的 ChatClient，用于与 AI 模型交互
    private final ChatClient chatClient;

    // 通过构造函数注入 ChatClient.Builder
    public RefundService(ChatClient.Builder chatClientBuilder) {
        this.chatClient = chatClientBuilder.build();
    }
}
```

**关键点**:
- `@Service`: 标记为 Spring 服务组件，可被自动扫描和注入
- `Logger`: 用于记录 AI 对话过程，实现行为可观测性
- `ChatClient`: Spring AI 核心类，封装了与 LLM 的交互

### 1.2 Prompt 模板设计

```java
private static final String SYSTEM_PROMPT = """
    # Role
    你是一名专业的电商平台客户体验专家...
    
    # Task
    请严格遵循以下步骤与用户进行对话：
    
    第一步：主动识别与确认
    - 识别关键词："根本没法用"、"是坏的"、"有瑕疵"...
    - 共情与确认：用封闭式问题确认问题
    
    第二步：判断与执行退款
    - 触发条件：用户确认质量问题
    - 立即行动：主动发起退款申请
    
    第三步：后续安抚与闭环
    - 表达歉意
    - 提供确定性
    
    # Limit
    - 仅处理质量问题
    - 不索要额外信息
""";
```

**设计思路**:
- 使用结构化 Prompt（Role-Task-Limit 框架）
- 明确 AI 的行为边界和决策流程
- 提供标准话术示例，确保回复一致性

### 1.3 对话方法实现

```java
public String chat(String userMessage, List<ChatMessage> history) {
    // 1. 记录用户输入（可观测性）
    log.info("[AI对话] 用户输入: {}", userMessage);
    
    // 2. 构建消息列表
    List<Message> messages = new ArrayList<>();
    
    // 系统消息：定义 AI 角色和行为
    messages.add(new SystemMessage(SYSTEM_PROMPT));
    
    // 历史消息：保持对话上下文
    for (ChatMessage msg : history) {
        if ("user".equals(msg.getRole())) {
            messages.add(new UserMessage(msg.getContent()));
        } else {
            messages.add(new AssistantMessage(msg.getContent()));
        }
    }
    
    // 当前用户消息
    messages.add(new UserMessage(userMessage));
    
    // 3. 调用 AI 模型
    long startTime = System.currentTimeMillis();
    String response = chatClient.prompt(new Prompt(messages))
            .call()           // 同步调用
            .content();       // 获取文本内容
    long endTime = System.currentTimeMillis();
    
    // 4. 记录 AI 响应和耗时
    log.info("[AI对话] AI响应: {}", response);
    log.info("[AI对话] 响应耗时: {}ms", endTime - startTime);
    
    return response;
}
```

**关键概念**:
- `SystemMessage`: 设置 AI 的系统级指令
- `UserMessage`: 用户输入的消息
- `AssistantMessage`: AI 的回复消息
- `Prompt`: 封装了完整的对话上下文

## 2. 控制器层 - RefundController

**文件路径**: `src/main/java/com/example/functioncall/controller/RefundController.java`

### 2.1 类定义

```javan@RestController              // 标记为 REST API 控制器
@RequestMapping("/api/refund")  // 基础路径
public class RefundController {
    
    private final RefundService refundService;
    
    // 简单的内存会话存储（生产环境应使用 Redis）
    private final Map<String, List<ChatMessage>> sessionStore = new HashMap<>();

    public RefundController(RefundService refundService) {
        this.refundService = refundService;
    }
}
```

### 2.2 对话接口

```java
@PostMapping("/chat")
public ChatResponse chat(@RequestBody ChatRequest request) {
    String sessionId = request.getSessionId();
    String userMessage = request.getMessage();
    
    // 记录请求日志
    log.info("[退款客服] 收到请求 - SessionId: {}, 消息: {}", 
             sessionId, userMessage);
    
    // 获取或创建会话历史
    List<ChatMessage> history = sessionStore.getOrDefault(
        sessionId, new ArrayList<>()
    );
    
    // 调用 AI 服务
    String aiResponse = refundService.chat(userMessage, history);
    
    // 保存对话历史
    history.add(new ChatMessage("user", userMessage));
    history.add(new ChatMessage("assistant", aiResponse));
    sessionStore.put(sessionId, history);
    
    // 记录响应日志
    log.info("[退款客服] 响应完成 - SessionId: {}, 历史轮数: {}", 
             sessionId, history.size() / 2);
    
    return new ChatResponse(aiResponse, sessionId);
}
```

**设计要点**:
- 使用 `sessionId` 维护多轮对话上下文
- 内存存储适合演示，生产环境需用 Redis
- 详细的日志记录便于调试和审计

### 2.3 重置会话接口

```java
@PostMapping("/reset")
public String resetSession(@RequestParam String sessionId) {
    sessionStore.remove(sessionId);
    log.info("[退款客服] 会话已重置 - SessionId: {}", sessionId);
    return "会话已重置";
}
```

// 生成或恢复会话 ID
let sessionId = localStorage.getItem('refundSessionId');

    sessionId = 'session_' + Date.now() + '_' + Math.random().toString(36).substr(2, 9);

    // 1. 显示用户消息

    const data = await response.json();
    
    // 4. 显示 AI 回复
    hideTypingIndicator();
    addMessage(data.response, 'assistant');
}

## 4. 日志配置 - logback-spring.xml

**文件路径**: `src/main/resources/logback-spring.xml`

### 4.1 配置详解

```xml
<configuration>
    <!-- 控制台输出：带颜色的日志 -->
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>
                %d{yyyy-MM-dd HH:mm:ss.SSS}  <!-- 时间戳 -->
                %highlight(%-5level)          <!-- 彩色日志级别 -->
                [%thread]                     <!-- 线程名 -->
                %cyan(%logger{36})            <!-- 青色 Logger 名 -->
                - %msg%n                      <!-- 消息内容 -->
            </pattern>
        </encoder>
    </appender>

    <!-- 文件输出：按时间和大小滚动 -->
    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>logs/function-call-demo.log</file>
        
        <!-- 
            SizeAndTimeBasedRollingPolicy:
            - 按天滚动（%d{yyyy-MM-dd}）
            - 按文件大小滚动（%i）
            - 最多保留30天
            - 总大小不超过3GB
        -->
        <rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
            <fileNamePattern>logs/function-call-demo.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
            <maxFileSize>100MB</maxFileSize>
            <maxHistory>30</maxHistory>
            <totalSizeCap>3GB</totalSizeCap>
        </rollingPolicy>
        
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} %-5level [%thread] %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>

    <!-- 业务日志：DEBUG 级别 -->
    <logger name="com.example.functioncall.service.RefundService" level="DEBUG"/>
    <logger name="com.example.functioncall.controller.RefundController" level="DEBUG"/>

    <!-- Spring AI 日志：查看 AI 调用细节 -->
    <logger name="org.springframework.ai" level="DEBUG"/>

    <!-- 根日志级别 -->
    <root level="INFO">
        <appender-ref ref="CONSOLE"/>
        <appender-ref ref="FILE"/>
    </root>
</configuration>
```

### 4.2 常见错误及修复

**错误**: `TimeBasedRollingPolicy` 不支持 `%i` 参数

```xml
<!-- 错误配置 -->
<rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
    <fileNamePattern>logs/xxx.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
    <maxFileSize>100MB</maxFileSize>  <!-- 不兼容！ -->
</rollingPolicy>

<!-- 正确配置 -->
<rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
    <fileNamePattern>logs/xxx.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
    <maxFileSize>100MB</maxFileSize>
</rollingPolicy>
```

| 技术点 | 说明 |

## 9. 扩展建议

1. **会话持久化**: 使用 Redis 替代内存存储
2. **限流保护**: 添加 RateLimiter 防止滥用
3. **敏感词过滤**: 对用户输入进行安全检查
4. **模型降级**: API 异常时返回兜底回复
5. **对话分析**: 收集数据优化 Prompt
